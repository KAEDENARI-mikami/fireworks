# Fireworks — 企画 / 要件定義書

> 作成日: 2026-06-16
> 講義課題: 時空間データベースを用いた Web アプリケーションの企画（課題7）
> 最初の成果物: 本企画を「インタラクティブレイアウトでアホほど綺麗」に見せる **単一 HTML ページ**（Bootstrap 可）。サーバへアップロードし URL を提出。

---

## 0. ひとことコンセプト

> **現実の場所に「会場」を設営し、戦い（弾幕）そのものが花火として夜を照らす。それを規定期間（最大1週間）、絶やさず守り切る放置ローグライク。**

観るべきは戦闘そのもの。会場設営は大げさな儀式ではなく、ただ舞台を組むだけ。そこで繰り広げられる弾幕の応酬が、そのまま美しい花火になる。

---

## 1. ジャンルと着想（3本の柱）

位置情報 × 放置タワーディフェンス × ローグライク。時空間データベースを核にした Web アプリ。

| 参照 | 持ち込む柱 |
|---|---|
| **Ingress** | 現実の場所＝会場。地図と街がゲーム盤。サイバーネオンの発光 UI |
| **東方 Project** | 戦闘は**弾幕**。敵 wave も撃退も幾何学的で美しい＝「綺麗な花火のような戦い」の正体 |
| **The Tower** | 核の**放置 × ローグライク TD**。中央拠点に押し寄せる wave、放置で進行、ラン単位、研究ツリーでメタ進行 |

---

## 2. 核となるゲームループ

```
会場設営（現実スポットを選択 → GPS座標 + 設置時刻を記録）
   ↓ 規定期間（24h / 3日 / 最大7日。長いほど高難度・高報酬）
放置中、その場所 × 時間帯のルールで弾幕 wave が断続的に襲来
   ↓ 基本は放置。時々チェックインして強化・修復（チェックインも時刻イベントとして記録）
規定期間を耐え切る → 点火成功（大輪の花火＝クリア演出）→ 報酬・図鑑・技術ツリーにメタ進行
   ↓（途中で陥落した場合）
run 失敗 → 会場ロスト（ローグライク）
```

**1周期（1プレイ）= 規定期間（最大1週間）、塔が現実時間を通して持ちこたえること。**

---

## 3. 見た目 / 世界観

- **方向性**: 近未来サイバーネオン。ただし「侵食の禍々しさ」ではなく、**撃退の瞬間が美しい花火のように咲く**祝祭的なトーン。
- **配色軸**: 黒 / 濃紺ベース + シアン・マゼンタ・ゴールドの発光。
- **戦闘表現**: 東方ライクの弾幕。光が花のように咲く粒子表現＝デモ映え最強。
- **デザイン目標**: 見た人に「美しさで S」を確定させるレベルの UI/UX。

---

## 4. 時空間データベースとの対応（講義要件①②③）

| 講義要件 | このアプリでの実現 |
|---|---|
| ① GPS からのデータ取得 | 会場設営時にブラウザ Geolocation API で座標取得 |
| ② 時空間データ管理 | 会場の座標＋設置時刻、wave 出現の「場所 × 時間帯」ルール、防衛戦の時系列（数日分の wave イベント + HP 推移 + チェックイン履歴）、ラン履歴の集積 |
| ③ 地図作成 | メイン画面が地図。場所別 / 時間帯別の戦績ヒートマップ、ランのリプレイ / タイムライン再生 |

**ポイント**: 単塔デプロイ型でも「1ラン = 1スポット」を積み上げることで「いつ・どこで・どんな成績だったか」の濃い時空間ログになる。1周期が数日に渡るため、1拠点あたりの時系列データも豊か。

## 6. スコープ（ビジョン → MVP → 発展）

### MVP（学期内に作り切る核）
- 現実スポットを1つ選択 → 会場設営（座標＋時刻記録）
- 規定期間（まずは短め: 24h など）耐久
- 簡易な弾幕 wave（場所 × 時間帯で内容が変わる）
- 生存判定（耐え切る / 陥落）
- 戦績を時空間 DB に記録、地図に履歴ピン表示

### 発展
- 期間バリエーション（3日 / 7日）と報酬カーブ
- 技術ツリー / 図鑑（ローグライクのメタ進行）
- 場所別・時間帯別の戦績ヒートマップ
- 防衛戦のリプレイ / タイムライン再生
- （将来構想）拠点ネットワーク型・陣取り対戦（Ingress フル）
