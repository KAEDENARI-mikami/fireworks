# 敵 wave スケーリング設計（2026-06-23）

## 目的
wave が進むほど敵が強くなる「The Tower 型」の恒久スケーリングを入れる。
背景：旧実装では `eRamp = min(1, 0.4+0.12*wave)` が wave5 でフラットになり、
**wave5 以降は敵1体あたり強くならず**（増えるのは湧き速度 `diff` のみ、wave140で頭打ち）、
規定 wave までヌルく流れてしまう穴があった。

## 参照（The Tower 実測 Tier1）
- W1: Damage 1.18 / HP 2.35
- W15000: Damage 4.63q(≈3.9e15) / HP 78.79ad(≈7.9e43)
- → 1waveあたり HP ×1.0067、攻撃 ×1.0024（HPは攻撃の約2.8倍速く伸びる）

## 決定事項
- **想定 wave 上限：15,000**（1wave=20秒 ＝ `WAVE_SECONDS`。放置で数日かけて押し上げる）
- **スケール率：HP・攻撃・ボス すべて `×1.0032^(wave-1)`**（単一レートで確定）
  - 約 **217wave で2倍**、W1000 ×24、W5000 ×866万、W10000 ×75兆、**W15000 ×6.4e20**
  - 速度は**据え置き**（理不尽回避。湧き数は既存 `diff` が担う）
  - 旧 `eRamp` は廃止し本カーブに置換。密度 `eLvMul(=1+総密度/300)`・色密度は残して掛け算
  - 実ステ = `基礎値 × eLvMul × 色密度 × 1.0032^(wave-1)`
- **数値表示ルール**：万・億・兆 までは日本語単位、**京以上（兆より上）は指数表記**（例 `6.4e20`）。京/垓は使わない。
  - ※ 表示フォーマッタ `fmt()` は未実装（次フェーズ）

## 実装（実装済み）
- `const SCALE_PER_WAVE = 1.0032; function waveMul(w){ return Math.pow(SCALE_PER_WAVE, Math.max(0,(w|0)-1)); }`
- 適用箇所：`spawnEnemy`（eRamp→wm）、`spawnOneBoss`、`summon`、要塞オービット(2000*eLvMul*wm)、掃討機の火の玉(500*eLvMul*wm)
- 決定論：wave は整数クロックなので `Math.pow` は純粋関数 → 前面＝裏(catch-up) byte一致を維持
- 検証(headless): waveMul(1000)=24.33、敵HP/攻撃とも wave1→1000 で ×24.33（一致）、FG==CU、JSエラー0

## 宿題（次フェーズ：プレイヤー経済ループ）
攻撃も ×6.4e20 まで伸びるため、**プレイヤーは火力だけでなく耐久（実効HP）も同規模で伸ばせる**必要がある（さもないと終盤即溶け）。
idle TD 定石：敵硬化→撃破$が指数増→火力&耐久を無限強化→着弾前に溶かす DPS レース。
- 撃破$報酬カーブ（現状 `cashEarned += (2+敵HP×0.18)×cashMul` で敵HP連動の土台あり）
- $アップグレードの無限レベル＆コスト傾斜（dmg/hp は現状 cap 無し）
- 耐久（HP/armor/dr/shield）の伸ばし方
- **桁表示 `fmt()`**（万/億/兆→指数）を HUD($/HP)・リザルト・資源に適用
