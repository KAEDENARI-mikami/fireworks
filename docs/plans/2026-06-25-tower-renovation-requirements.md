# Fireworks タワー改装 要件定義

2026-06-25

## 目的

タワー改装は、The Tower のモジュール相当を Fireworks 向けに再設計した恒久装備システムである。

既存のスキルはガチャで得る戦闘中の特殊能力、研究は歩行で進む長期解禁、恒常強化は資源による基礎値強化である。タワー改装はこれらと役割を分け、塔そのもののロードアウトを作る機能にする。

## 基本方針

- 初手から全カテゴリ解禁。
- 各カテゴリは「素体1枠 + 対応カテゴリのアドオン3枠」で構成する。
- 戦闘開始時に装備状態をスナップショットし、そのラン中は変更不可。
- 戦闘中に改装しても次ランから反映。
- 複数塔運用時は、将来的に塔ごとロードアウトへ拡張できる構造にする。
- ガチャ、歩き、戦闘のどれでも進む。
- すべての成果を無駄にしない。

## カテゴリ

### 砲台

攻撃担当。通常DPSの中核。

- ダメージ
- 連射
- 射程
- 多重弾
- クリティカル
- 貫通
- 対ボス火力

### バリア装置

防御とバリア担当。瞬間耐久と被弾対策。

- バリア量
- シールド
- バリア回復
- 被弾無効
- 反射
- バリア破壊時効果

### 装甲

HPと防御担当。基礎耐久の土台。

- 最大HP
- 防御力
- 防御率
- 再生
- 被ダメ軽減
- 連続被弾耐性

### 発信装置

スキルとトリッキー系担当。戦闘ルールを書き換える枠。

- スキル威力
- チャージ速度
- 敵減速
- 脆弱化
- wave操作
- 敵増加で報酬増
- HP条件効果

### リアクター

自由枠。ビルドの中心。

- $獲得
- 資源採取
- SPARK
- 黒色火薬補助
- 火力変換
- 防御変換
- リスク報酬型の固有効果

## モジュール構造

モジュールは、カテゴリごとの素体と3つのアドオンで完成する。

```text
カテゴリモジュール = 素体 + アドオン1 + アドオン2 + アドオン3
```

## 素体

素体はカテゴリの基幹パーツ。

- 保有特殊スキル、またはカテゴリ固有効果を持つ。
- 基本的なアドオン強化倍率を持つ。
- ガチャや高難度報酬で入手する。
- 既存の「モジュール」は、この素体として扱う。

```js
{
  uid,
  defId,
  category,
  rarity,
  level,
  stars,
  addonMul,
  uniqueEffect,
  equipped
}
```

## アドオン

アドオンは道端や戦闘中で拾う小型パーツ。

```js
{
  uid,
  category,
  rarity,
  mainEffect,
  subEffects,
  lockedSubCount,
  equippedSlot
}
```

- 対応するカテゴリにのみ装着可能。
- 各カテゴリ最大3つ装着可能。
- メイン効果を1つ持つ。
- レア度に応じて最大5つのサブ効果を持つ。
- メイン効果とサブ効果は、既存ステータスのうち1つを強化する。
- サブ効果自体にもレアリティがあり、レアリティに応じて数値が伸びる。

## アドオンのサブ効果数

- Rare: サブ1つ
- Epic: サブ2つ
- Legend: サブ3つ
- Mythic: サブ4つ
- Celestial: サブ5つ

## サブ効果リロール

アドオン分解で得るトークンを使い、サブ効果をリロールできる。

```text
リロール費用 = (保持するサブ効果数 * 2 + 1)^2 * リロールするサブ効果数 * 10
```

例:

- 何も保持せず5つリロール: `(0*2+1)^2 * 5 * 10 = 50`
- 2つ保持して3つリロール: `(2*2+1)^2 * 3 * 10 = 750`
- 4つ保持して1つリロール: `(4*2+1)^2 * 1 * 10 = 810`

保持数が増えるほど狙い撃ちが高価になる。

## 入手経路

### ガチャ

黒色火薬を使う即時入手ルート。

- 通常改装箱: 黒色火薬25。素体を入手。
- カテゴリ指定箱: 黒色火薬60。選択カテゴリの素体を入手。
- 高級箱: 将来実装。Legend以上保証ゲージ向け。

位置づけは時短と補填。最強装備の主入手手段にはしない。

### 歩き

位置ゲームとしての継続保証。

- 500mごと: アドオントークン。
- 2kmごと: Rare以上アドオン抽選。
- 初訪問セル、スイートスポット発見、長距離歩行は将来拡張。

歩けば最低でも素材が増えるため、移動が無駄にならない。

### 戦闘

本命の入手ルート。

- ボス撃破: アドオントークン、低確率でアドオン。
- 到達wave精算: 敗北、撤退、踏破のすべてでアドオントークン。
- スイートスポット踏破: 高レアアドオン箱を将来追加。

戦闘に負けても到達waveに応じて資産が残る。

## 無駄なし経済

- 不要素体は分解して改装片とカテゴリ片へ変換。
- 不要アドオンは分解してリロール用トークンへ変換。
- 低レアも分解、星上げ素材、図鑑ボーナス候補として価値を残す。
- ガチャは天井進捗を持つ。
- 最高レアが出なくても次の保証へ近づく。
- 戦闘失敗でも到達wave報酬を得る。
- 歩行でも必ず素材が貯まる。

## MVP実装範囲

- 改装画面。
- 5カテゴリ各1素体 + 各3アドオン枠。
- 素体の所持、装備、解除、分解。
- アドオンの所持、装備、解除、分解。
- サブ効果保持数の指定とリロール。
- 黒色火薬25の通常改装箱。
- 黒色火薬60のカテゴリ指定箱。
- 改装片とカテゴリ片。
- アドオントークン。
- 10回以内にEpic以上保証。
- 40回以内にLegend以上保証。
- 歩行500mトークン、2kmアドオン。
- ボス撃破トークンと低確率アドオンドロップ。
- リザルト到達waveトークン。
- 戦闘開始時の改装ステータス反映。

## 実装上の制約

- 現状は単一HTML、localStorage保存を維持。
- 戦闘再生の決定論を壊さないため、ラン中の素体とアドオン状態は `saveRun` に保存する。
- 改装変更は進行中ランへ即時反映しない。
- 乱数は既存の決定論PRNGか、メタ報酬用の保存seedを使う。
- 既存のスイートスポット難度モジュールとは別物として扱う。
