## 概要

現実の場所に「会場」を1つ設営し、中央の塔1基が360°から押し寄せる敵wave（ドローン編隊）を**全自動**で迎撃する放置TD。プレイヤーは照準もよけもせず、唯一の能動操作は**黄スキル（起動式アルティメット）**の編成と発火のみ。戦闘の見た目はすべて**夜空の花火＋ドローンショー**で、秩序（intact＝編隊シルエット）vs 混沌（被弾・撃破＝花火バースト）の視覚文法で統一する（"ignition／点火"はメカではなく純粋な演出）。

The Tower 系譜（Black Hole / Death Wave / Chain Lightning の再スキン、放置進行、ラン単位、研究ツリー）を核に、Ingress風の**位置＝資源フィールド**を最大の差別化点とする。設置スポットは5色密度ベクトルを持ち、**総密度∝難易度（Greed=Danger）**。判定はハイブリッド：規定期間（最大7日）生存＝**踏破（クリア）ボーナス**、到達wave／キル数＝**SCORE＆報酬**、HP=0＝**陥落**。

## コアループ

```
① 地図でスポット選択 → GPS座標+設置時刻を記録、5色密度ベクトルと TimeBox(24h/72h/168h) を確認
② 初期ドラフト：提示3枚から コア塔モジュール+開幕カード2枚 を選ぶ（ローグライク）
③ 放置開始：時刻からwave番号を決定論的に算出（オフラインでも進行）。time-of-dayで密度が"呼吸"
④ 放置中：5色を密度比でアイドル採取。能動操作は黄スキル起動のみ
⑤ チェックイン：採取資源を消費して塔をビルド/修復、ドラフトでカード追加（時刻イベントとして記録）
⑥ TimeBox満了 → 踏破（大輪フィナーレ演出）→ 報酬精算 → メタ進行
   ／ HP=0 → 陥落（会場ロスト）。ただし到達waveぶんのSCORE・Sparkは確定精算
```

**1ラン = 1スポット × 最大7日のリアルタイム。** どこに置くか（WHERE）が採取できる色＝可能なビルドを規定し、いつチェックインするか（WHEN）が強化・修復・スキル温存の戦略になる。全色潤沢なJACKPOTを7日踏破＝**聖地**。

## 場所＝資源フィールド（5色×濃度）

- 現実空間を **H3セル（res≈9, 約75m）** でグリッド化。1セル=1スポット。初回GPS踏破時にlazy生成し、**基準5色密度ベクトル D=(赤,青,緑,黄,紫)**（各0–100）を確定。
- **採取（idle-harvest）**: `採取/分(color) = base × D[color]/50`。D=50で等倍、D=100で2倍。採取色は対応ビルドカテゴリ「のみ」に使える（色＝カテゴリ固定）。
- **密度の呼吸（time-of-day）**: 1日4帯（朝05–10 / 昼10–16 / 夕16–19 / 夜19–05）で色別係数が変動。代表傾向：
  - 紫・黄：夜ピーク（×1.4 / 昼×0.7）── 夜は稼ぎ時かつ修羅場
  - 赤：夕ピーク（×1.25）／青：昼安定高（×1.15）／緑：朝ピーク（×1.3）
- **総密度＝難易度**: `TotalD = ΣD`（0–500）。共通乗数 `densityMul = 0.6 + TotalD/250`（TotalD=100→1.0、250→1.6、500→2.6）が spawnCount・敵HP・速度に乗る。豊かさと危険が完全比例。
- **分布**: 通常スポットは1–2色リッチ（該当色60–90、残り5–30）の「偏った会場」。全色70+のJACKPOTは世界の数%。
- 偏りの救済は**紫の資源変換**とメタ進行（初期ランク／採取効率底上げ）が担い、立地は「最適化要素であって必須条件にしない」。

**wave構造**: 1 wave=90秒（交戦60s+IV30s）。`spawnCount = baseN × densityMul × surgeMul`、`baseN = 8 + floor(waveNo/40)`。10wave毎にラッシュ×1.8、50wave毎に大ラッシュ×3.0。**楽章**：序章(1–120)→第1(121–480)→第2(481–960=1日)。以降960wave毎に**深度D+1**（最大7）。

## メタ進行・経済

**二層経済**: 5色資源はラン内に閉じ、メタ層は単一 **Spark**（+稀少 **Relic**）に集約。

- **ラン内**: 採取色は対応ツリーのみに充当。緑が採取率を複利化、紫が偏りスポットの不足色を変換補填（ロス付き）。色ノンストック＝「採れた色でしか組めない＝WHEREがビルドを決める」原則を担保。
- **leftover→meta**: ラン終了時（踏破/陥落とも）残資源を**10:1でSparkに精錬**。一方向流出。
- **Spark獲得式**: `SCORE = round(8×√reach_wave) × (踏破1.5倍) × greed係数(1+0.12×密度総量)`。例：到達70踏破密6→287 / 到達150踏破密25→980 / 陥落40密4→75。
- **Relic**: 初回踏破実績（初72h／初JACKPOT／各色特化踏破）と図鑑コンプ節目のみ。Tier昇格と最上位LAB・伝説モジュールに必須＝進行の天井を律する。

**恒久成長4レイヤー**
- **(A) LAB（放置研究／時間次元連動）**: Sparkで着手し**現実経過時間**で完成。tier1=2h/40Spark … tier4=72h/1500+Relic1。同時スロット1→最大3。効果は全ランへ恒久適用（採取率+、初期HP+、スキルスロット+、ドラフト枚数3→4）。
- **(B) モジュール**: コア塔の恒久パーツ。各色1枠+汎用2枠=計7スロット。ランごと組み替え＝ビルドの骨格。
- **(C) カードプール**: ドラフト母集団をメタで拡張・昇格。
- **(D) 図鑑ZUKAN**: 敵編隊・銘・スポット種別・time-of-day変種を収集。ラン履歴がそのまま図鑑ソース。

**プレステージ**: スポット解禁Tier（I 偏り24h → II 複数色リッチ72h → III 全色準リッチ → IV JACKPOT/聖地168h）。境界昇格にSpark+Relic。任意リセット『観測者リセット』で永続乗算ボーナス（リブート回数×8%）。

## 時空間データ設計（要件①②③）

**2層モデル**: 静的な場（FIELD＝世界の真実＝資源密度）と、動的なラン（RUN＝時系列イベント）。

| テーブル | 役割 | 主キー時空間列 |
|---|---|---|
| `spot` | H3セル1区画（lazy生成） | center/cell GEOGRAPHY(4326), base_density NUMERIC[5], total_density/is_jackpot GENERATED |
| `spot_density_profile` | 時間帯別密度（呼吸） | PK(spot_id,color,tod_bucket), density, harvest_rate |
| `run` | 防衛戦1回（最大7日の親） | deploy_point GEOGRAPHY, planned_hours, deployed/ended_at, tstzrange GENERATED, outcome, reached_wave, harvest_total[5] |
| `run_tick` | 等間隔スナップ（リプレイ/グラフ） | PK(run_id,seq), tower_hp, wave_no, harvest_cum[5], tod_bucket |
| `run_event` | 離散点（checkin/skill_fire/build/boss/fall） | t, kind, actor, at_point GEOGRAPHY, payload JSONB |
| `lab_research` | 放置研究 | started_at, complete_at GENERATED(=started+duration) |
| `player`/`meta_upgrade`/`collection_entry` | メタ進行・図鑑 | first_seen_at/_spot_id（初発見も時空間ログ） |

- **① GPS取得**: デプロイ確定時に `navigator.geolocation` で実測座標+精度を `run.deploy_point` / `deploy_accuracy_m` へ。生GPS点を H3セル（`spot`）にスナップし、生点とセルの両方を残す。
- **② 時空間管理**: 空間=全GEOGRAPHY(4326)+GIST（ST_DWithin/ST_Contains/ST_AsMVT）。時間=tstzrange＋run_tick等間隔時系列（7日×5分=最大~2016行/ran）＋run_event離散列＋tod_bucket＋lab経過時間。run（粗）→run_tick（中）→run_event（細）の**多重解像度ログ**。
- **③ 地図作成**: ①5色ヒートマップ（ST_AsMVT、色レイヤ切替、total_dで難易度マップ化）②JACKPOT発見ビュー（is_jackpot抽出＋初発見の名誉）③日夜アニメ（tod_bucketタイムスライダで密度脈動）④ランリプレイ（run_tick時間再生＋run_event重畳＋レーダーチャート）。

**技術**: PostgreSQL16+PostGIS3（Supabaseホスティング）、run_tickはTimescaleDBハイパーテーブル+continuous aggregate、H3（res≈9）、5色NUMERIC[5]、Mapbox GL（ダーク）+deck.gl、Supabase Realtimeでライブ観戦、pg_cronでlab完了判定・タイル更新。戦闘描画はCanvas/PixiJS or Three.js（index.html流用）でデータ層と疎結合。

## 数値の初期当て（叩き台）

| 項目 | 値 |
|---|---|
| 採取レート | `base × D[color]/50 /分`（D=50等倍, D=100で2倍） |
| 黄採取 | 黄密度1.0で約60黄/時（7日理論上限~10,000黄） |
| wave周期 | 90秒（交戦60s+IV30s）。1時間≈40wave / 1日≈960wave / 7日≈6,700wave |
| 物量 | `spawnCount = (8+floor(waveNo/40)) × densityMul × surgeMul` |
| 共通乗数 | densityMul=0.6+TotalD/250 / depthMul=1.0+0.5×(D-1) / waveMul=1.0+0.003×waveNo(cap6.0) |
| 敵HP | `baseHP × densityMul × depthMul × (1+0.002×waveNo)`。基本敵HP10/18/22/35/45/80 |
| 塔初期HP | MaxHP=1000（青Lv25で4000、Lv50で7000） |
| 塔基礎火力 | dmg10 × RoF0.8発/秒 × 1発 × crit5%（射程半径100m相当） |
| ボスHP | `4000 × 3.0^(D-1) × (1+TotalD/5)`、制限8分 |
| Spark式 | `round(8×√reach) × 踏破1.5 × (1+0.12×密度総量)` |
| バランス目安 | 低TotalD~120=24h向け / 中200–300=3日適正 / 聖地350+=7日が壁（実効HP×18相当, フルビルド+スキル回し前提） |

数値インフレ対策の共通ガード：乗算軸（RoF×Multishot×CritPower×Pierce、緑の複利、黄の割合・無限スケール）には全て**明確なキャップ**（Multishot最大8、DR上限60%、レゾナンス+120%、貫通/跳弾の逓減）を置き、`densityMul`の総密度比例苛烈化で報酬と危険の帳尻を合わせる。描画は弾プール・命中バースト上限（粒子MAX≈4200）で制御。

## MVPスコープ

**学期内に作り切る核**（既存企画書§6準拠）
- スポット1つ選択 → 会場設営（GPS座標+時刻記録、`spot`/`run` INSERT）
- 規定期間（まず**24h**）耐久。waveは時刻から決定論算出
- 簡易弾幕wave（場所×時間帯で内容変化＝densityMul + tod_bucketのみ）
- 5色採取 → ビルドは**赤（火力数本）+ 青（HP/Regen）+ 黄（連雷1銘）**に絞る
- 紫は**重力坩堝+減速+連鎖アーク+錬成炉の4ノード**程度に限定（多層実装は重いため）
- 生存判定（踏破/陥落）、SCORE/Spark精算
- 戦績を時空間DBに記録（run_tick等間隔+run_event）、地図に履歴ピン+5色ヒートマップ1枚
- 提出物の課題7本体＝**ビジョンを「アホほど綺麗」に見せる単一HTML**（index.html＝3Dタイトル画面を流用）。スキーマ実体（migrations）はMVPでは未着手でも可

**発展**: 72h/168h報酬カーブ、全ツリー解禁、ボス『大輪』、LAB/図鑑/モジュール、日夜アニメ・リプレイ、聖地ランキング、ライブ観戦。

## 決定事項（2026-06-16 確定）

### 敵 ＝ 資源フィールドの鏡像（重要・新規）

スポットの5色密度ベクトルは、プレイヤーのビルドだけでなく**敵のステータスにも対称に効く**。*その土地のエネルギーは、塔も敵も等しく武装させる*。

- **色別密度 → 対応する敵ステータス上昇**（プレイヤーの色＝カテゴリの鏡像）:
  - 🔴赤密度 → 敵の**攻撃力**↑ ／ 🔵青密度 → 敵の**HP・耐久**↑ ／ 🟢緑密度 → 敵の**バフ・自己強化**↑ ／ 🟡黄密度 → 敵の**スキル（特殊行動）**↑ ／ 🟣紫密度 → 敵の**トリッキー（妨害・回避・分裂等）**↑
  - → 赤リッチな会場は「殴り合い」、青リッチは「硬い壁の波」、紫リッチは「搦め手の地獄」… **立地が敵の"性格"を決める**。
- **総密度 → 敵の初期レベル上昇**（旧 `densityMul` を初期Lvに読み替え）。総密度が高いほど開幕から高Lvの編隊。JACKPOT＝高Lv × 全ステ盛り＝**聖地の地獄**。
- 旨味：自分が強くしたい色（採れる色）が、そのまま敵の同系統も強くする＝**同じ土俵での殴り合い**。色選択＝得意分野で敵と張り合う構図。

### 操作・スキル

- **手動 ⇄ 放置でパフォーマンス不変**：在席しても性能は上がらない。PROGRAM のオート発火で常に最適に回る。手動発火は"演出・触り心地"であって有利不利なし＝**完全な放置ゲー**。
- **スキルは攻撃型に限らない**：自動／手動／**パッシブ（常時発動の恒常効果）**まで含む多型（例：不死鳥＝パッシブ復活、祝祭の譜＝バフ場）。
- **黄ゲージは上限なし・減衰なし**：ため込み可（終夜大輪等の溜め撃ちを許容）。

### 経済（2層・更新 / 旧案を上書き）

- **ラン中の強化 ＝ $（敵を倒すと入る）**。撃破で `$ += ceil(2 + 敵maxHP × 0.18)`、$ で5系統（攻撃/防御/採取/スキル/減速）をラン内強化（毎ラン リセット）。The Tower の coin 強化に相当。
- **5色資源 ＝ 恒常的な強化（メタ）専用**。立地の色別濃度で**放置採取**し、持ち帰って永続強化（LAB／モジュール／解禁）に使う。**ラン内ビルドには使わない**。
- → 旧「5色資源はラン内ツリーに充当／leftover→Spark」は撤回。立地＝**メタ資源の稼ぎ場＋難易度**、ラン内ビルド＝**$（実力＝撃破数）**で駆動。緑の「採取」強化は $ を払って恒常資源の採取量を増やす＝メタ投資の橋渡し。

### ラン長・運用

- **時間圧縮はしない。** 代わりに**実時間5分の「5分モード」**を用意（短い実防衛＝デモ・クイックプレイ兼用）。TimeBox ＝ **5min / 24h / 72h / 168h**。評価・デモは5分モードで実施。
- 紫の資源変換ロス：**1.0 → 0.5 固定**（立地の意味を守る側）。
- time-of-day 基準：**会場の経度から算出**（再現性・公平）。
- 数値カーブ：**新カーブ（総密度→初期Lv、色密度→該当ステ、depth × 線形）に一本化**。旧指数カーブ（10×1.18^w）は破棄。
- 緑の遅咲き救済：**入れない**（緑は遅咲きエンジンのまま）。
- MVP の DB：提出は**単一HTML**主、Supabase に**最小スキーマのみ**（本実装は発展）。

## 未決事項（当初案・上記「決定事項」で解決済み）

1. **手動発火 vs 放置の格差**: 大火/百花繚乱/終夜大輪/流転は在席前提。PROGRAMのオートキャストで救済する設計だが、最大値が在席者に偏る。チェックイン頻度の想定（1日1–2回？）とオフライン処理の上限をどこに置くか。
2. **「ため込み」最適解の抑制**: 終夜大輪に全黄を温存して「7日何もせず溜める」が最適化されないよう、黄ゲージ上限・減衰・即時発火報酬のどれで縛るか。
3. **紫・錬成炉の変換ロス率**: 強すぎると「位置＝資源フィールド」という核の差別化が薄れる。初期0.5起点で固定か、もっと厳しくするか（立地の意味とビルド自由度のトレードオフ）。
4. **緑の遅咲き/青依存の強制度**: 緑単色不可は意図的だが、序盤陥落しやすい。低Tierに即効採取ブーストをどこまで置いて初心者を救うか。
5. **time-of-dayの基準時刻**: 端末ローカル時刻 か 会場経度から算出 か（夜偏重メカ＝紫黄ピークの公平性とデモ再現性に直結）。
6. **TimeBoxの実時間運用**: 最大7日は講義デモ・採点に対し長すぎる。デモ用に時間圧縮モード（例: 1分=1時間）を別途用意するか、24hランのみで評価するか。
7. **数値カーブの最終整合**: 既存企画書の `敵HP≈10×1.18^(w-1)`（wave100で1.3億）と、本仕様の `densityMul×depthMul×(1+0.002×waveNo)` 線形寄りカーブが**二重定義で衝突**。どちらを正とするか（指数だとwave100到達は事実上不可能、線形だと7日6,700waveまで伸びる）── 要一本化。
8. **MVPのDB実体**: migrations未作成。課題7提出は単一HTMLのみで、DB設計は紙面（本仕様）に留めるか、Supabaseで最小スキーマだけ立てるか。
