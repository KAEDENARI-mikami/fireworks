# Fireworks ── 位置情報モード ＆ 決定論放置シム 要件定義

> 2026-06-21 / 課題7（時空間DB Webアプリ）／ The Tower 寄せの放置タワーディフェンス × 位置情報

本書は、既存プロトタイプ（`index.html`：3D戦闘＋拠点＋強化＋出撃準備）に対し、**出撃準備画面を「位置情報マップ」に置き換え**、戦闘を **The Tower 型の無限ウェーブ放置（決定論）** に作り替えるための要件を確定する。設計書 `2026-06-16-fireworks-game-design.md` の時空間DB青写真（H3セル・5色密度・time-of-day呼吸・2層モデル）を土台に、実装スコープを以下に絞る。

---

## 0. スコープと前提

- **現状はプレ（試作）**。ステータス・ボス・戦闘数値は本実装で大きく変わる前提。本要件は「戦闘の**枠組み**（無限ウェーブ・決定論・放置）」と「**位置レイヤー**」を定義し、戦闘内部は疎結合に差し替え可能とする。
- **敵ロスターは現状維持**（mote / halo / spiral / shooter ＋ 重戦車・掃討機・機動要塞）。今回は種類を増やさない。
- **単一HTML・ビルドレス維持**。外部依存はCDN（three.js と同様）。
- **DBは極力使わない**。場（フィールド）は手続き生成で完全に再現できるため永続化不要。永続するのはメタ進行とアクティブ・ラン状態のみ（localStorage）。

---

## 1. 位置情報モード（出撃準備の置き換え）

一般的な位置情報ゲーム（Pokémon GO / Ingress）と同じ体感。

- **地図**：Leaflet ＋ ダークタイル（CDN・トークン不要）をデフォルト。既存のサイバー・ネオン基調に合わせる。
- **現在地**：`navigator.geolocation`（実GPS）で地図を追従、自分＝アバター表示。**PC/開発検証用にアバターを手動移動できる疑似モード**を併設（机では歩けないため）。セキュアコンテキスト（https or localhost）必須という制約を明記。
- **世界＝H3セルグリッド**（res≈9, 約75m）。**どこに立っても、その場所のセルの5色密度で戦える**。歩く＝別セル＝別フィールド。
- **密度は手続き生成（DBなし・決定論）**：`density = f(cellId, tod_bucket)`。`hash(cellId)` ＋ time-of-day 係数で算出。**同じ場所はいつ来ても同傾向**（良い場所を覚えて再訪できる）。サーバ不要。
- **スイートスポット**＝同じハッシュから稀に湧く高密度セル（＝JACKPOT）。地図上で強調表示。出所はDBでなく手続き判定。
- **出撃＝今いるセル**でのみ可能（その場に物理的にいる必要がある＝位置ゲーの核）。出撃すると、そのセルの5色密度ベクトルが**ランのフィールド値**として戦闘へ受け渡される。
- 旧「密度スライダー＋ボス手動選択」は廃止。フィールド（難易度・色プロファイル）はセルが決める。**ボスの決まり方は戦闘内部の論点として据え置き**（既定案：セルの色プロファイルで種別、超高密度JACKPOT＝ALL。調整可）。

## 2. 戦闘＝The Tower 型・無限ウェーブ放置（決定論）

- **時間制限を撤廃**。ウェーブを**規定ウェーブまで無限に自動進行**。
- **終了条件**：規定ウェーブ到達＝**踏破（クリア）** ／ 塔陥落（HP0）＝**陥落**。
- **放置進行**：アプリを閉じても進む想定。実時間スケールは**最終的に1ラン＝最大1週間**で規定ウェーブ到達（調整可）。

### 2.1 「裏でも完全に同じ計算」＝決定論シミュレーション

ユーザー要件：放置中も近似でなく**まったく同一の戦闘計算**。閉じている間はブラウザは計算しないため、実体は「**起動時に経過ぶんを再構築**」となる。これを近似ナシで満たすため：

- **真実＝決定論シム一本**：戦闘を `(出撃時刻・セルID・5色密度・ビルド・シード)` から**完全再現可能な純粋関数**にする。前面でも放置追いつきでも、回すのは**同一の `runStep`**。結果は完全一致。
- **固定タイムステップ**（dtを固定）。可変dtや実フレーム時刻への依存を排除。
- **シード付きPRNG**：`Math.random()` を全廃し、シード付き決定論PRNGに置換（再現性の根幹＝本実装の土台工事）。
- **前面**：実時間でステップ＋3D描画。
- **起動時（不在ぶんあり）**：`経過時間 ÷ 固定dt = 飛ばすステップ数` を、**同じ `runStep`（描画なし）でチャンク分割 or Web Workerで非同期に追いつき**。その間プレイヤーは拠点・地図・強化を**普通に操作可能**。隅に進捗（例「追いつき中… wave 312 / 経過4日」）。現在に追いつく or 規定ウェーブ or 陥落で停止。
- **セーブ＝状態の最小核**：`(出撃時刻・セルID・シード・現在tick・ビルド・到達ウェーブ・HP 等)`。場は手続き生成なので保存不要。

### 2.2 正直な技術的但し書き

- 1週間まるごと放置→起動は数百万〜1千万ステップ規模になり得る。非同期でも実時間で数秒〜数分 churn する可能性。**UIは生かしたまま**進め、体験は壊さない。
- 万一プロファイルで重すぎる場合の保険：**「演出は間引くが数字は同一」忠実度ノブ**を残す（既定は完全同一）。最終手段として、真実の計算単位を*フレーム*から*ウェーブ*へ上げる「ウェーブ単位決定論リゾルバ」も理論上の代替として記録（ただし現方針は毎ステップ同一）。

## 3. 土台工事（本実装で必須）

1. **シード付きPRNG導入**：`rand()`／`Math.random()` の全使用箇所をシード駆動に置換。スポーン角度・乱数CD・クリティカル判定・パーティクル等、**戦闘結果に影響する乱数はすべて**シード化（純演出パーティクルは別系統の非決定乱数でも可だが、結果不変なら分離）。
2. **dt固定 ＋ ループ刷新**：可変dtを廃し固定dt。前面は実時間ペーシング、追いつきは最速チャンク。
3. **無限ウェーブ化**：5分タイマー／assault→boss の固定フェーズを、規定ウェーブまでのウェーブ進行＋ボスウェーブ（既存ボスを節目に配置）へ再構成。
4. **状態機械の更新**：`deploy`（出撃準備）→ `map`（位置マップ）へ置換。`title / home(拠点) / map / playing / result` ＋ 放置追いつきオーバーレイ。
5. **永続化**：localStorage に meta（恒常進行）＋ active run（決定論セーブ核）。

## 4. アーキテクチャ契約（疎結合）

```
セル(緯度経度→cellId) ──f(cellId, tod)──▶ フィールド値 { 5色密度, seed, 派生難易度 }
        │
        ▼
   ラン = 決定論シム( フィールド値, ビルド )  ← 単一の真実
        │                                   ├─ 前面: runStep×実時間 + 3D描画(可視化)
        │                                   └─ 放置: runStep×経過tick（描画なし・非同期追いつき）
        ▼
   終了(規定ウェーブ=踏破 / HP0=陥落) → 報酬精算 → メタ強化 → 次のセルへ
```

- **戦闘内部（敵編成・ボス・数値）は箱**。入力＝フィールド値、出力＝ラン結果（到達ウェーブ・キル・資源・踏破/陥落）。中身は後日刷新可能。
- **描画は真実の可視化**であり、真実そのものではない（決定論シムが正典）。

## 5. 据え置き／今後の決定

- ボスの決まり方（セル色プロファイル → 種別、JACKPOT → ALL の既定案で進めるが要調整）。
- スイートスポットの寿命・呼吸（既定：静的＝同じ場所は常に当たり、time-of-day で密度のみ脈動）。
- 陣取り／マルチプレイ（MVP対象外）。
- 時空間DBの本格利用（課題評価向けの履歴ヒートマップ等）は、まず手続き生成＋localStorageで動かしてから検討。
